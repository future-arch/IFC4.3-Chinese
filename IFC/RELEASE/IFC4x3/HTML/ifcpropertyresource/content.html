8.16 IfcPropertyResource

8.16.1 架构定义

IfcPropertyResource 定义了一组基本的属性类型，这些属性可以通过 IfcPropertySet（在 IfcKernel schema 中定义）与实例对象（IfcObject 的子类型）和类型对象（IfcTypeObject 的子类型）关联。这些属性用于分配属性定义，这些定义由名称字符串在语义上定义，并且在定义方面是可扩展的。

注释 注释 经常需要扩展附加到单个对象或对象组的特性。然而，对于同一类中的每个对象，可能没有必要扩展特性。使用与元素类型相同的功能，可以定义此类特性的集合并将它们与单个对象关联。

特性的对象关联可以通过两种不同的方式定义：

  • 通过定义一个类型对象（IfcTypeObject），该对象定义了一个特定的对象信息（包括相关属性），并通过 IfcRelDefinesByType 关系分配给该类型对象的所有实例（或出现）。
  • 通过定义一个属性集（IfcPropertySet，相关属性的逻辑分组），该属性集定义了单个实例（或出现）的通用属性信息，这是通过 IfcRelDefinesByProperties 关系完成的。

单个属性所表示的语义信息必须通过 Name 属性给出。Name 是人类可读的，也可以通过约定被计算机解释。这种约定必须由实施者协议建立。

注释 注释 IFC 规范定义了一组属性集，为 IFC 模型中的对象提供了属性集的推荐用法。除了 IFC 属性集规范外，还可以进行区域或项目特定的属性集定义。因此，属性集提供了一种本地化 IFC 定义的机制。

IfcProperty 的所有子类型稍后都将由 IfcPropertySet 引用。因此，IfcPropertySet 通过 IfcSimpleProperty（及其子类型）和 IfcComplexProperty 建立了一个属性定义的树。此树中的 IfcSimpleProperty 的任何实例只能被引用一次（通过 IfcPropertySet 或 IfcComplexProperty）。

注释 注释 附加到对象的属性集应具有树形结构，因此所有属性（IfcSimpleProperty -及其子类型- 或 IfcComplexProperty）在该结构中只能使用一次。
