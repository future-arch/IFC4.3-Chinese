IFC"建筑与设施管理行业数据共享工业基础类"是一个开放的BIM数据国际标准，用于在建筑或设施管理行业各参与方使用的软件应用程序之间交换和共享。该标准包含的定义涵盖了建筑和基础设施工程在其生命周期内所需的数据。 现已纳入IFC的基础设施设施范围包括桥梁、道路、铁路、水路和港口设施。

IFC包括数据模式的发布、其文档、属性和数量集定义以及交换文件格式结构的机制。 该模式、属性和数量集以及使用约束在内部以UML类图形式编写 可作为XMI文件获取 并发布为计算机可解释模式，包括：

  • 根据ISO 10303-11，采用EXPRESS数据规范语言
  • 根据ISO 10303-28，采用XML模式定义语言(XSD)
  • 作为RDF/OWL模式（开发中）
  • 作为JavaScript对象表示法模式（开发中）
  • 作为实体、预定义类型和属性的JSON结构化分类法
注释 有关模式的发布，请参见附录A"计算机可解释列表"。

根据计算机可解释模式交换和共享数据的交换文件格式包括：

  • 交换结构的明文编码，定义于ISO 10303-21
  • 可扩展标记语言(XML)，定义于XML Schema W3C建议书，具有IFC特定转换
  • RDF/OWL，由W3C和buildingSMART项目定义
  • JSON JavaScript对象表示法，由buildingSMART项目定义

文件格式的概述可在技术网站上查看。如果符合数据模式要求，也可以使用其他交换文件格式。

完整的IFC标准用于定义建筑和设施管理行业中的一个或多个认可的工作流程。IFC数据模式和引用数据的此类交换场景定义被称为模型视图定义(MVD)。每个MVD为开发软件应用程序确定数据交换要求。符合要求的软件应用程序在申请软件认证时需要说明其符合的模型视图定义。

通用用途

以下内容属于此IFC发布版本的范围：

  1. 建筑和基础设施生命周期各阶段所需的BIM交换格式定义：

    • 需求论证
    • 需求构思
    • 初步可行性研究
    • 实质性可行性研究和初步财务审批
    • 初步概念设计
    • 完整概念设计
    • 协调设计
    • 采购和完整财务审批
    • 生产信息
    • 施工
    • 运营和维护

  2. 生命周期各阶段涉及的各专业所需的BIM交换格式定义：

    • 建筑和土木工程设计
    • 服务和公用设施工程
    • 结构工程
    • 采购
    • 施工规划
    • 设施和公用设施管理
    • 项目管理
    • 客户需求管理
    • 行业主管部门的许可和审批

  3. 包括以下内容的BIM交换格式定义：

    • 项目结构
    • 物理组件
    • 空间组件
    • 分析项
    • 流程
    • 资源
    • 控制
    • 参与者
    • 上下文定义
