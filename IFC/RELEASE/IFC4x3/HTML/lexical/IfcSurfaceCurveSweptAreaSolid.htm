8.8.3.41 IfcSurfaceCurveSweptAreaSolid

注释 该实体不是标准化模式子集或实现级别的一部分。

8.8.3.41.1 语义定义（Semantic definition）

IfcSurfaceCurveSweptAreaSolid 是将一个区域沿位于参考曲面上的导曲线扫掠的结果。扫掠区域由 IfcProfileDef 的子类型提供。轮廓通过隐式笛卡尔变换算子在扫描的起始点放置，此时轮廓法线与该点的导曲线切线一致，轮廓的 x 轴与该点的曲面法线一致。在导曲线的任何点上，扫掠轮廓的原点位于导曲线上，轮廓的法线指向导曲线的切线方向，轮廓的 x 轴与该点处的参考曲面法线相同。

注释 轮廓区域的法线必须在任何给定点与导曲线的切线相同。如果导曲线在起始点具有线性段，则该段必须垂直于起始处的轮廓。

Directrix 和 ReferenceSurface 在对象坐标系中定位。扫描操作的起始点是 StartParam，参数值基于曲线参数化提供。如果未提供 StartParam，则默认值为导曲线的开头。扫描操作的结束点是 EndParam，参数值基于曲线参数化提供。如果未提供 EndParam，则默认值为导曲线的末尾。实体的几何形状不依赖于曲线参数化；体积取决于扫掠的面积和 Directrix 的长度。

注释 StartParam 和 EndParam 默认情况下不是标准化的，它们取决于曲线的参数化。但是，使用 IfcCompositeCurve 中的 IfcReparametrisedCompositeCurveSegment 作为导曲线允许显式地重新参数化底层扫掠曲线。对于导曲线（如 IfcCircle 或 IfcEllipse）的闭合曲线，StartParam 和 EndParam 不得超过参数范围，不得大于 360°。
示例 参考曲面是位于三维空间中的任何曲面（平面、圆柱面、复合曲面），并放置在对象坐标系中。在大多数情况下，它是平面或挤出曲面。导曲线位于曲面上，如果是平面，则可以定义为折线或复合曲线；如果是圆柱面或其他非平面参考曲面，则通常可以定义为该参考曲面上的 p 曲线。

在导曲线的任何点上，都可以构造一个平面。隐式平面的位置坐标系的原点位于导曲线上。位置坐标系的 Axis3（z 轴或法线）与该点的导曲线切线相同，位置坐标系的 Axis1（x 轴或 u）与参考曲面法线相同。Axis2（y 轴或 v）被构造出来。

如果省略继承的 Position 属性，则扫掠实体的所得主体不会被重新定位。否则，由 Position 属性建立的坐标系将用于相对于对象坐标系重新定位主体。

示例 图 8.8.3.41.A 说明了一个使用圆柱参考曲面和用于扫掠矩形的 p 曲线的示例。未提供 Position，因此它不会重新定位所得的扫掠实体。图 8.8.3.41.B 显示了预期的结果。
注释 导曲线的起点位于对象坐标系的原点，如图所示，这只是巧合。导曲线的起点，从而扫描操作的起点，可能位于对象坐标系内的任何点，并且仅取决于导曲线的位置。
注释 定义根据 ISO/CD 10303-42:1992 曲面曲线扫掠区域实体是一种扫掠区域实体，它是将一个面沿位于 ReferenceSurface 上的 Directrix 扫掠的结果。SweptArea 的方向与法线方向相关。
注释 实体的定义改编自 ISO 10303-42 中定义的 surface_curve_swept_area_solid。
沿革 新实体在 IFC2x2 中引入。

SweptArea 要求是位于 z = 0 平面内的有界曲线曲面，并沿 Directrix 扫掠，使得用于定义 SweptArea 的局部坐标系的原点位于 Directrix 上，并且局部 x 轴沿当前点的 ReferenceSurface 的法线方向。所得实体的特性是，通过 Directrix 上任意点的法平面与 SweptArea 是相同的副本。

SweptArea 沿 Directrix 扫掠的方向由一个三维笛卡尔变换算子精确定义，其属性为： > * 局部原点为点 (0., 0., 0)， * 轴 1 为参数 u 的 Directrix 点处的参考曲面的法线 N。 * 轴 3 为参数 u 的 Directrix 点处的切线向量 t 的方向。其余属性默认为定义一个随导曲线参数 u 变化的相应变换矩阵 T(u)。

Informal Propositions

  1. SweptArea 应位于隐式平面 z = 0 内。
  2. Directrix 应位于 ReferenceSurface 上。
