8.18.3.4 IfcGeometricRepresentationContext

RV ⓘ
AbV ⓘ

8.18.3.4.1 语义定义（Semantic definition）

IfcGeometricRepresentationContext 定义了适用于项目中多个产品形状表示的上下文。它定义了定义形状表示的上下文类型以及此上下文中定义的几何表示项的数值精度。此外，它还可以使用 WorldCoordinateSystem 属性来偏移项目坐标系与全局原点。主表示上下文还可以提供真北方向，请参见图 8.18.3.4.A。

如果 WorldCoordinateSystem 的 y 轴不指向全局北向，则应提供 TrueNorth 属性。真北方向或地理北向相对于 WorldCoordinateSystem 属性建立的项目坐标系统的方向。它由项目坐标系统内的二维方向给出。如果未提供，则默认为 [0.,1.] - 即项目坐标系统的正 Y 轴等于地理北向。方向在项目坐标系统中提供，并标识真北方向。

图 8.18.3.4.A 中的“CAD 中看到的真北”显示了 TrueNorth 方向的正确解释。

如果通过 HasCoordinateOperation 属性包含到地理坐标系统的转换，则应省略 TrueNorth 属性或仅将其包含在信息目的上。它不应添加到 IfcCoordinateOperation 已应用的任何变换中。

使用一个 IfcGeometricRepresentationContext 实例来表示模型（3D）视图是强制性的，使用第二个 IfcGeometricRepresentationContext 实例来表示平面（2D）视图是可选的（但如果存在比例相关的平面视图，则需要给出），其他比例或视图相关的上下文需要通过使用指向模型视图（或平面视图）作为 ParentContext 的子类型 IfcGeometricRepresentationSubContext 来处理。请参见图 8.18.3.4.B，其中显示了使用几何表示上下文表示 3D 和 2D，并分配了子上下文。

注释 The inherited attribute ContextType shall have one of the following recognized values: 'Model', 'Plan', 'NotDefined'.

为 3D 模型和 2D 平面上下文定义的表示上下文，包括不同目标比例的子上下文定义。对于 2D 和 3D 坐标空间，始终应只有一个几何表示上下文。

注释 Definition according to ISO/CD 10303-42:1992 A geometric representation context is a representation context in which the geometric representation items are geometrically founded.
注释 Entity adapted from geometric_representation_context defined in ISO 10303-42.
沿革 New entity in IFC2.0.
IFC2x3 变更 Applicable values for ContextType are only 'Model', 'Plan', and 'NotDefined'. All other contexts are now handled by IfcGeometricRepresentationSubContext.
IFC4 变更 Mapping to Geographic coordinate system added by inverse attribute HasCoordinateOperation.

8.18.3.4.5 形式化表示（Formal representation） 

ENTITY IfcGeometricRepresentationContext
 SUPERTYPE OF (ONEOF
	(IfcGeometricRepresentationSubContext))
 SUBTYPE OF (IfcRepresentationContext);
	CoordinateSpaceDimension : IfcDimensionCount;
	Precision : OPTIONAL IfcReal;
	WorldCoordinateSystem : IfcAxis2Placement;
	TrueNorth : OPTIONAL IfcDirection;
 INVERSE
	HasSubContexts : SET [0:?] OF IfcGeometricRepresentationSubContext FOR ParentContext;
	HasCoordinateOperation : SET [0:1] OF IfcCoordinateOperation FOR SourceCRS;
 WHERE
	North2D : NOT(EXISTS(TrueNorth)) OR (HIINDEX(TrueNorth.DirectionRatios) = 2);
END_ENTITY;
