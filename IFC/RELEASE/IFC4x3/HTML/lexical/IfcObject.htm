5.1.3.6 IfcObject（对象）

RV ⓘ
AbV ⓘ

抽象 该定义可能不会被直接实例化。

5.1.3.6.1 语义定义（Semantic definition）

IfcObject 是对任何经过语义处理的事物或过程的泛化。对象即事物呈现的形式——即实例。

注释 IfcObject 的示例包括物理实体（如墙、梁或覆盖层）、物理存在的项目（如空间）或概念性项目（如网格或虚拟边界）。它也代表过程（如工作任务）、控制项（如成本项）或参与者（如设计过程中涉及的人员）。

对象可通过继承的 Name 属性进行命名，该名称应为用户可识别的对象实例标签。可使用继承的 Description 属性对对象进行进一步说明。ObjectType 属性的用途包括：

  • 当存在 PredefinedType 属性且其值设为 USERDEFINED 时，为 IfcObject 所有子类型存储用户定义值
  • 在未提供 PredefinedType 属性的情况下，提供 IfcObject 子类型的类型信息（可视为极轻量级分类器）。这在缺乏预定义类型完整列表时尤为常见

对象是独立的信息单元，可包含或引用其他信息单元。对象可能涉及以下几种关系：

  • 与外部/内部资源信息的关联 - 指向外部/内部信息源的关联关系。更多信息请参阅超类型 IfcObjectDefinition
  • 其他对象的分配 - 指向其他类型对象的分配关系。更多信息请参阅超类型 IfcObjectDefinition
  • 其他对象的聚合 - 建立整体/部分关系的聚合关系。对象既可以是整体，也可以是部分，或同时兼具两种角色。更多信息请参阅超类型 IfcObjectDefinition
  • 类型的分配：IsTypedBy - 通过 IfcRelDefinesByType 定义关系，使用类型定义来规定此实例的通用特征，可能包括分配给该类型的所有对象实例的通用形状表示和通用属性。这是一种具有隐含依赖关系的特定-实例关系（实例属性依赖于类型属性，但可覆盖它们）
  • 部分类型的分配：IsDeclaredBy、Declares - 通过 IfcRelDefinesByObject 定义关系，使用类型定义的组件（称为"声明部分"）来定义实例的组件（称为"反映部分"）。这也被称为"深拷贝"。实例中所有部件的通用特征由类型中的部件定义。这是一种具有隐含依赖关系的特定-实例关系（实例属性依赖于类型属性，但可覆盖它们）
  • 属性集的分配：IsDefinedBy - 通过 IfcRelDefinesByProperties 定义关系，将属性集定义分配给对象实例
注释 关于说明性图示请参见 IfcRelDefinesByType。同时可参阅如何通过实例属性覆盖类型属性。关于部分类型分配的说明性图示请参见 IfcRelDefinesByObject
沿革 在IFC1.0中新增实体
IFC4 变更 新增逆关系 Declares、IsDeclaredBy 和 IsTypedBy。类型不再包含在 IsDefinesBy 关系中。IfcProject 已升级为 IfcObjectDefinition -> IfcContext 的子类型

非正式命题

  1. 部分类型分配（即逆属性 IsDeclaredBy 或 Declares）仅当对象是分解的一部分时才应使用，即仅当存在 IsDecomposedBy 或 Decomposes 时使用

5.1.3.6.6 形式化表示（Formal representation） 

ENTITY IfcObject
 ABSTRACT SUPERTYPE OF (ONEOF
	(IfcActor
	,IfcControl
	,IfcGroup
	,IfcProcess
	,IfcProduct
	,IfcResource))
 SUBTYPE OF (IfcObjectDefinition);
	ObjectType : OPTIONAL IfcLabel;
 INVERSE
	IsDeclaredBy : SET [0:1] OF IfcRelDefinesByObject FOR RelatedObjects;
	Declares : SET [0:?] OF IfcRelDefinesByObject FOR RelatingObject;
	IsTypedBy : SET [0:1] OF IfcRelDefinesByType FOR RelatedObjects;
	IsDefinedBy : SET [0:?] OF IfcRelDefinesByProperties FOR RelatedObjects;
 WHERE
	UniquePropertySetNames : ((SIZEOF(IsDefinedBy) = 0) OR IfcUniqueDefinitionNames(IsDefinedBy));
END_ENTITY;
