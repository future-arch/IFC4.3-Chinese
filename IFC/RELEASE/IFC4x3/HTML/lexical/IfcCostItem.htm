6.5.3.2 IfcCostItem（成本项）

注释 该实体不是标准化模式子集或实现级别的一部分。

6.5.3.2.1 语义定义（Semantic definition）

IfcCostItem（成本项）用于描述成本或财务价值，以及描述其背景信息的说明性内容，使其能够在成本计划中使用。IfcCostItem（成本项）可用于表示商品和服务的成本、通过流程执行工作的成本、生命周期成本等。

每个 IfcCostItem（成本项）实例可以具有名称和描述。根据成本用途的不同，这些值应基于协议进行声明。例如，Name（名称）属性可用于提供一个通用值，使得不同的实例能够在嵌套安排中组合在一起（见下文），而 Description（描述）属性可用于提供成本计划中项目描述所使用的文本。

IfcCostItem（成本项）可以链接一个或多个 IfcCostValue（成本值），这些值表示单位成本、总成本，或一个单位成本与一个或多个用于生成总成本的数量。数量可以作为单独的数量给出，或者这些数量由一个或多个建筑元素作为元素数量提供。IfcCostValue.Category（成本值类别）属性表示成本的类别，可用于在特定列中呈现该值。对于嵌套成本项（具有 IfcRelNests 关系），IfcCostValue.Category（成本值类别）非常重要，使得 IfcCostValue.AppliedValue（成本值应用值）计算为所有具有相同 IfcCostValue.Category（成本值类别）的嵌套成本之和，或者如果设置为星号（'*'），则计算所有成本类型的所有嵌套成本之和。IfcCostValue（成本值）可以表示原始值或由公式导出的值。例如，税费可以计算为小计的百分比。

沿革 IFC2.0 中的新实体。
IFC4 变更 添加了属性 PredefinedType（预定义类型）、CostValues（成本值）和 CostQuantities（成本数量）。

6.5.3.2.5 形式化表示（Formal representation） 

ENTITY IfcCostItem
 SUBTYPE OF (IfcControl);
	PredefinedType : OPTIONAL IfcCostItemTypeEnum;
	CostValues : OPTIONAL LIST [1:?] OF IfcCostValue;
	CostQuantities : OPTIONAL LIST [1:?] OF IfcPhysicalQuantity;
END_ENTITY;
