8.16.3.6 IfcPropertyBoundedValue

RV ⓘ
AbV ⓘ

8.16.3.6.1 语义定义（Semantic definition）

具有边界值的属性 IfcPropertyBoundedValue 定义了一个属性对象，该对象最多有两个（数值或描述性）值，第一个值指定上限，第二个值指定下限。它定义了一个属性-值边界（最小-最大）组合，其中给出了属性 Name、可选的 Description、可选的 UpperBoundValue（带度量类型）、可选的 LowerBoundValue（带度量类型）以及可选的 Unit。除了上限和下限值之外，还可以提供一个设定点值用于操作值设置。

单位由 Unit 特性处理，表 8.16.3.6.A 显示了边界属性的示例：

  • 如果未提供 Unit 特性，则单位已由 IfcMeasureValue 或 IfcDerivedMeasureValue 的类型隐含。关联单位可在项目级别全局定义的 IfcUnitAssignment (IfcProject.UnitsInContext) 中找到。
  • 如果提供了 Unit 特性，则 Unit 特性分配的单位将覆盖全局分配的单位。

IfcPropertyBoundedValue 允许为属性描述的值分量指定一个区间。如果未提供 LowerBoundValue 或 UpperBoundValue，则表示一个开放边界（最小值或最大值）。区间定义为包含性的，即 LowerBoundValue 或 UpperBoundValue 中给定的值包含在区间内。

注释 An IfcPropertyBoundedValue may be exchanged with no values assigned yet. In this case the LowerBoundValue and the UpperBoundValue are set to NIL.
沿革 IFC2x 中的新实体。

IFC2x2 变更：UpperBoundValue 和 LowerBoundValue 特性的类型已设为可选，并保持文件交换的向上兼容性。

IFC4 变更：添加了 SetPointValue 特性。

非正式命题

  1. 如果上限值和下限值的度量类型是数值度量，则以下条件必须成立：UpperBoundValue > LowerBoundValue。

8.16.3.6.5 形式化表示（Formal representation） 

ENTITY IfcPropertyBoundedValue
 SUBTYPE OF (IfcSimpleProperty);
	UpperBoundValue : OPTIONAL IfcValue;
	LowerBoundValue : OPTIONAL IfcValue;
	Unit : OPTIONAL IfcUnit;
	SetPointValue : OPTIONAL IfcValue;
 WHERE
	SameUnitLowerSet : NOT(EXISTS(LowerBoundValue)) OR NOT(EXISTS(SetPointValue)) OR
(TYPEOF(LowerBoundValue) = TYPEOF(SetPointValue));
	SameUnitUpperLower : NOT(EXISTS(UpperBoundValue)) OR NOT(EXISTS(LowerBoundValue)) OR
(TYPEOF(UpperBoundValue) = TYPEOF(LowerBoundValue));
	SameUnitUpperSet : NOT(EXISTS(UpperBoundValue)) OR NOT(EXISTS(SetPointValue)) OR
(TYPEOF(UpperBoundValue) = TYPEOF(SetPointValue));
END_ENTITY;
