5.4.3.63 IfcSite（场地）

注释 该实体是 IfcProduct 或 IfcTypeProduct 的子类型，因此是每个标准化模式子集和实现级别的一部分。

5.4.3.63.1 语义定义（Semantic definition）

场地是指项目施工将要完成的特定陆地区域，可能包含水域。场地可用于建造、改造或拆除建筑物，或用于其他与施工相关的开发。

注释 根据ISO 6707-1的定义：进行建筑工程或其他开发活动的陆地或水域区域。

场地的精确地理空间信息应通过将_IfcProject_与现实世界相关联的地理参照实体来获取，使用基准数据集（参见_IfcCoordinateOperation_和_IfcCoordinateReferenceSystem_）。

在小型项目（如建筑项目）中，当未按上述建议提供地理参照时，可使用_IfcSite_的_经度_、纬度_和_高程_属性来近似指示场地位置。此方法并非要取代精确的地理参照，但对于不需要精确地理空间信息的用例（如遮阳模拟）仍然有用。如果已声明，_经度、_纬度_和_高程_将确定_IfcSite_的_LocalPlacement_的点0.,0.,0.在WGS84中的位置。

由_IfcLocalPlacement_定义的场地几何放置应始终相对于包含此场地的空间结构元素，或是绝对的，即相对于项目的几何表示上下文所建立的世界坐标系。在_IfcProject.RepresentationContexts_中建立的世界坐标系可能包括世界坐标系XY平面内的真北定义，如果提供，可以在_IfcGeometricRepresentationContext.TrueNorth_中获取。

一个项目可能跨越多个相连或不相连的场地。因此，场地复合体提供了项目中包含的场地集合。一个场地也可以分解为多个部分，每个部分定义一个场地段。这是由超类型_IfcSpatialStructureElement_的组合类型属性定义的，解释如下：

  • COMPLEX = 场地复合体
  • ELEMENT = 场地
  • PARTIAL = 场地段

图SITECOMP显示了_IfcSite_作为空间结构的一部分。除了逻辑空间结构外，还显示了放置层次结构。在此示例中，空间结构层次结构和放置层次结构是相同的。

注释 关于强制性元素包含和放置结构关系的详细要求在视图定义和实施者协议中给出。

图SITELOC描述了_IfcSite_的高度和高程。它用于提供场地原点的地理经度、纬度和海拔高度。场地的原点是局部放置。

在_IfcSite_中提供经度、纬度和高度用于地理参照是为了向上兼容。它需要_IfcSite_的单个实例和WGS84作为坐标参照系。

对于精确的地理参照（或引用除WSG84之外的任何其他地理坐标系），必须使用实体_IfcCoordinateReferenceSystem_和_IfcMapConversion_来定义项目工程坐标系到地理（或地图）坐标系的精确映射。

  • 场地的参考高程由以下提供：IfcSite.RefElevation，根据该位置使用的高程基准给出。
  • 位于场地的每个建筑物的参考高程均相对于该位置使用的相同高程基准给出。
  • 属于每个建筑物的每个楼层的标高作为相对于建筑物参考高程的局部高度给出。
沿革 IFC1.0中的新实体。

5.4.3.63.5 概念用法（Concept usage）

5.4.3.63.6 形式化表示（Formal representation） 

ENTITY IfcSite
 SUBTYPE OF (IfcSpatialStructureElement);
	RefLatitude : OPTIONAL IfcCompoundPlaneAngleMeasure;
	RefLongitude : OPTIONAL IfcCompoundPlaneAngleMeasure;
	RefElevation : OPTIONAL IfcLengthMeasure;
	LandTitleNumber : OPTIONAL IfcLabel;
	SiteAddress : OPTIONAL IfcPostalAddress;
END_ENTITY;
