5.3.3.6 IfcTask（任务）

RV ⓘ
AbV ⓘ

5.3.3.6.1 语义定义（Semantic definition）

IfcTask 是在建筑项目中需要执行的可识别工作单元。

任务通常用于描述产品或安装的施工活动，但不限于这些类型。例如，它也可用于描述设计过程、搬迁操作以及其他设计、施工和运营相关活动。

任务消耗的资源数量通过为资源定义 IfcElementQuantity 来处理，而不是在 IfcTask 实例中处理。

沿革 IFC1.0 中的新实体。在 IFC2x 中从 IfcWorkTask 重命名。
IFC4 变更 添加了 TaskTime 和 PredefinedType 属性。IfcMove 和 IfcOrderRequest 在 IFC4 中已被移除，现在由 IfcTask 表示。IfcRelAssignsTasks 关系也已被移除。

5.3.3.6.1.1 Attribute use definition

每个 IfcTask 实例都被赋予一个指示其内容的名称（IfcRoot.Name）。可以提供任务的文本描述，并且可以通过叙述性长描述（IfcProcess.LongDescription）进一步阐述。可以为执行任务所使用的工作方法声明工作方法。任务被标识为里程碑任务或非里程碑任务。里程碑任务由标记 IsMilestone 定义，并且没有持续时间。还可以设置每个任务的状态和优先级。

5.3.3.6.1.2 Time and duration use definition

与之前的 IFC 版本相比，基本的任务时间信息（计划开始时间、计划完成时间、持续时间）现在通过 TaskTime 属性直接附加到 IfcTask。常规任务通过 IfcTaskTime 定义。重复性任务通过 IfcTaskTimeRecurring 定义。如果常规任务是从重复性任务派生的，则这两个任务应通过 IfcRelNests 关系链接在一起，其中 IfcRelNests.IsNestedBy 指向重复性任务，IfcRelNests.Nests 指向从重复性任务派生的所有常规任务。

5.3.3.6.1.3 Representation of other activities

IfcTask 的使用定义已泛化以表示其他活动，包括在之前的 IFC 版本中由自有实体定义的活动。这包括：

  • 订单操作
  • 搬迁操作

IfcTask 表示一个订单，可能由帮助台执行，该帮助台充当组织在设施用户与满足其需求的功能要求之间的接口角色。由 IfcTask 实体表示的实际任务是将请求转化为订单并启动使订单能够完成的行动。IfcProjectOrder 或其子类型之一（包括维护工作订单）使用 IfcRelAssignsToControl 与 IfcTask 相关联。

IfcTask 也可用于描述将人员、组织内的团体或整个组织及其相关的家具和设备从一个地方移动到另一个地方的活动。因此，它取代了之前的 IFC 实体 IfcMove。该功能在 IfcTask 中表示如下：

  • 迁出地：行动者及其相关设备迁出的地点。使用 IfcRelAssignsToProcess，其中 RelatingProcess 指向任务，RelatedObjects 保存要迁出的位置。
  • 迁入地：行动者及其相关设备迁入的地点。使用 IfcRelAssignsToProduct，其中 RelatedObjects 指向任务，RelatingProduct 指向要迁入的位置。
  • 整改清单：关于搬迁需要关注的事项列表。使用 LongDescription，或者通过 IfcRelNests 识别子任务以单独跟踪整改清单项目。

5.3.3.6.6 概念用法（Concept usage）

5.3.3.6.7 形式化表示（Formal representation） 

ENTITY IfcTask
 SUBTYPE OF (IfcProcess);
	Status : OPTIONAL IfcLabel;
	WorkMethod : OPTIONAL IfcLabel;
	IsMilestone : IfcBoolean;
	Priority : OPTIONAL IfcInteger;
	TaskTime : OPTIONAL IfcTaskTime;
	PredefinedType : OPTIONAL IfcTaskTypeEnum;
 WHERE
	CorrectPredefinedType : NOT(EXISTS(PredefinedType)) OR (PredefinedType <> IfcTaskTypeEnum.USERDEFINED) OR ((PredefinedType = IfcTaskTypeEnum.USERDEFINED) AND EXISTS(SELF\IfcObject.ObjectType));
	HasName : EXISTS(SELF\IfcRoot.Name);
END_ENTITY;
