5.3.3.5 IfcRelSequence（关系序列）

注释 该实体不是标准化模式子集或实现级别的一部分。

5.3.3.5.1 语义定义（Semantic definition）

IfcRelSequence 是过程之间的顺序关系，其中一个过程必须在另一个过程之前发生，且该关系的时间特性可描述为某种序列类型。关联过程（IfcRelSequence.RelatingProcess）被视为关系中的前驱（具有优先性），而被关联过程（IfcRelSequence.RelatedProcess）则是后继。

IfcRelSequence 被定义为一对一关系，因此它将一个前驱分配给一个后继。

沿革 IFC1.0 中的新实体。
IFC4 变更 已移至 IfcProcessExtension 模式。 TimeLag 和 SequenceType 设为可选。 USERDEFINED 已添加到 IfcSequenceEnum 枚举中。 添加了 UserDefinedSequenceType 属性。 添加了控制 USERDEFINED 枚举使用的 WHERE 规则。

使用定义

IfcRelSequence 用于描述两个过程之间存在的逻辑顺序关系。此逻辑关系标识存在一个前驱或关联过程以及一个后继或被关联过程。在 IFC 中，关系中可以有一个前驱和一个后继。可能存在多个 IfcRelSequence 实例来描述一个前驱任务与多个后继任务或多个前驱任务与一个后继任务的顺序关系，从而实现任务之间的多对多顺序关系。请注意，顺序关系既可用于定义过程实例之间的依赖关系，也可用于定义过程类型之间的依赖关系（更多信息请参见 IfcRelDefinesByObject 和 IfcTaskType）。在定义过程实例之间的依赖关系时，顺序关系应保持在有向无环图的限制范围内。

可以为序列设置序列类型。对于分配给工作进度计划的任务，预期将断言序列类型。对于过程图，其中过程之间的顺序关系仅仅是逻辑流，则无需断言。

可以为序列分配时间滞后，序列类型定义了时间滞后应用于序列的方式，可以是作为持续时间的比率或百分比（例如，当前驱任务完成 50% 时启动后继任务），也可以是作为时间度量（例如，在前驱任务开始一周后启动后继任务）。在根据序列类型的设置向序列分配时间滞后时应谨慎，因为不会检查时间滞后值是否与设置的序列类型一致。

5.3.3.5.6 形式化表示（Formal representation） 

ENTITY IfcRelSequence
 SUBTYPE OF (IfcRelConnects);
	RelatingProcess : IfcProcess;
	RelatedProcess : IfcProcess;
	TimeLag : OPTIONAL IfcLagTime;
	SequenceType : OPTIONAL IfcSequenceEnum;
	UserDefinedSequenceType : OPTIONAL IfcLabel;
 WHERE
	AvoidInconsistentSequence : RelatingProcess :<>: RelatedProcess;
	CorrectSequenceType : (SequenceType <> IfcSequenceEnum.USERDEFINED) OR ((SequenceType = IfcSequenceEnum.USERDEFINED) AND EXISTS(UserDefinedSequenceType));
END_ENTITY;
