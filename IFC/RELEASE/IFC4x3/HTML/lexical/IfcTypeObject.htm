5.1.3.46 IfcTypeObject（对象类型）

RV ⓘ
AbV ⓘ

5.1.3.46.1 语义定义（Semantic definition）

对象类型定义了关于某个类型的特定信息，这些信息对于该类型的所有实例都是通用的。它对应于公认的_通用-特定-实例_建模范式中的特定层级。IfcTypeObject 通过 IfcRelDefinesByType 关系分配给各个对象实例（即实例）。

注释 术语"类型"和"样式"经常互换使用。

对象类型通过一组属性集定义来表示。附加的属性集描述了对象类型可用的字母数字信息，并用于定义适用于该类型所有对象实例的通用属性。

注释 如果在分配给 IfcTypeObject（及其子类型）的 IfcPropertySet 和该类型的实例中使用了具有相同名称的属性，则实例属性将覆盖类型属性。有关说明性图示，请参见 IfcRelDefinesByType。

对象类型可以在尚未分配给对象的情况下进行交换。对象类型可以指示其定义来源的库（或目录）。此关联通过继承的 HasAssociations 关系处理，该关系指向 IfcRelAssociatesLibrary。

沿革 IFC2x 中的新实体

IFC2x3 变更：IfcTypeObject 现在从新的超类型 IfcObjectDefinition 子类型化，并且属性 HasPropertySets 已从 LIST 更改为 SET。

IFC4 变更：从 IFC4 开始，实体 IfcTypeObject 不应被实例化。在未来的 IFC 版本中，它将更改为 ABSTRACT 超类型。反向属性 Types 已从 ObjectTypeOf 重命名。

5.1.3.46.6 形式化表示（Formal representation） 

ENTITY IfcTypeObject
 SUPERTYPE OF (ONEOF
	(IfcTypeProcess
	,IfcTypeProduct
	,IfcTypeResource))
 SUBTYPE OF (IfcObjectDefinition);
	ApplicableOccurrence : OPTIONAL IfcIdentifier;
	HasPropertySets : OPTIONAL SET [1:?] OF IfcPropertySetDefinition;
 INVERSE
	Types : SET [0:1] OF IfcRelDefinesByType FOR RelatingType;
 WHERE
	NameRequired : EXISTS(SELF\IfcRoot.Name);
	UniquePropertySetNames : (NOT(EXISTS(HasPropertySets))) OR IfcUniquePropertySetNames(HasPropertySets);
END_ENTITY;
