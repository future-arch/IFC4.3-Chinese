8.15.3.17 IfcProfileDef（轮廓定义）

RV ⓘ
AbV ⓘ

8.15.3.17.1 语义定义（Semantic definition）

IfcProfileDef 是IFC中所有标准和任意轮廓定义的超类型。它用于通过参数或显式曲线几何来定义一组常用的标准截面轮廓。

  • 参数化轮廓是二维图元，在行业中通过参数描述来定义横截面。
  • 任意轮廓是通过外边界（有界曲线）定义的横截面，也可能包括由内边界定义的孔洞。
  • 基于父轮廓变换的派生轮廓也是可用轮廓定义的一部分。
  • 此外，可以定义复合轮廓，其中包含两个或多个轮廓定义来定义最终轮廓。
沿革 在IFC1.5中新增实体，在IFC2x中功能已扩展。轮廓现在支持带有内边界的扫描曲面和扫描实体。它已从IfcAttDrivenProfileDef重命名。
IFC4 变更 对于不需要显式定义几何图形的用途，从ABSTRACT更改为非抽象。新增了逆属性_HasProperties_和_HasExternalReference_。

在材料关联中的使用

梁、柱和类似形状的建筑元素及其类型对象可以通过_IfcRelAssociatesMaterial_与_IfcMaterialProfileSet_和_IfcMaterialProfileSetUsage_结合，与截面轮廓定义和材料定义相关联。这样，具有相同截面和材料的建筑元素和元素类型可以共享通用的截面轮廓定义和关联。

材料关联中的轮廓定义需要与相应建筑元素的形状表示保持一致。

构件截面轮廓的空间对齐的更高级描述（如居中、左下、几何质心等）可以通过_IfcMaterialProfileSetUsage_中的基点参考来提供。这可以冗余于几何数据使用，以传达设计意图。

在形状模型中的使用

轮廓定义在几何和几何模型资源中用于创建扫描曲面、扫描实体或截面脊线。

在扫描曲面或扫描实体中使用轮廓定义的目的是定义一个被扫描的均匀横截面：

  • 沿直线（拉伸）使用_IfcSurfaceOfLinearExtrusion_或_IfcExtrudedAreaSolid_
  • 沿圆弧（旋转）使用_IfcSurfaceOfRevolution_或_IfcRevolvedAreaSolid_
  • 沿参考曲面上的准线使用_IfcSurfaceCurveSweptAreaSolid_

在截面脊线中使用轮廓定义的目的是沿着脊线曲线在多个位置定义变化的横截面。子类型_IfcDerivedProfileDef_特别适合提供基于起始轮廓变换的连续轮廓，从而保持顶点和边的标识。

注释 _IfcProfileDef_的子类型包含参数化轮廓（作为_IfcParameterizedProfileDef_的子类型），它们建立自己的2D位置坐标系；通过显式曲线几何给出的轮廓（开放或封闭轮廓）；以及两种特殊类型：基于2D笛卡尔变换的复合轮廓和派生轮廓。

当_IfcProfileDef_在扫描实体中使用时，它被视为有界区域。在这种情况下，轮廓的内部是轮廓的一部分。属性_ProfileType_设置为AREA。当_IfcProfileDef_在扫描曲面中使用时，它被视为曲线。在这种情况下，轮廓的内部（如果曲线闭合）不是轮廓的一部分。属性_ProfileType_设置为CURVE。

图1和图2说明了在扫描实体中使用参数化轮廓的示例。

IfcProfileDef在基础坐标系中定义，该坐标系由使用轮廓定义的扫描曲面或扫描实体定义。它是xy平面：

  • IfcSweptSurface.Position的xy平面，或
  • IfcSweptAreaSolid.Position的xy平面，或
  • IfcSectionedSpine.CrossSectionPositions每个列表成员的xy平面。

在图1左侧，位置坐标系的z轴指向绘图平面外。

注释 子类型IfcParameterizedProfileDef可选地提供相对于IfcProfileDef基础坐标系的额外2D位置坐标系。

在后续在扫描曲面或扫描实体中使用IfcProfileDef时，例如IfcExtrudedAreaSolid（此处用作示例），轮廓边界（此处基于IfcParameterizedProfileDef的2D位置坐标系）被放置在扫描曲面或扫描实体的3D位置坐标系的xy平面中。

轮廓通过以下方式插入基础坐标系：

  • 直接插入（使用IfcArbitraryClosedProfileDef和IfcArbitraryOpenProfileDef时）
  • 通过中间位置坐标系插入（使用IfcParameterizedProfileDef时）
  • 通过2D笛卡尔变换运算符插入（使用IfcDerivedProfileDef时，对于任意轮廓定义直接应用于曲线位置，对于参数化轮廓定义应用于位置坐标系）
  • 使用IfcCompositeProfileDef时，插入方式取决于包含的子轮廓的子类型

轮廓类型

此处展示了_ProfileType_属性不同使用方式的结果。_ProfileType_定义内部（有界区域）是否是轮廓定义的一部分（Area）或不是（Curve）。图3和图4说明了根据_ProfileType_产生的区域或曲线。

通过外部引用指定轮廓

如果轮廓通过规范或目录标准化，应通过_HasExternalReference_提供对此规范或目录的引用。此逆关系用于将_IfcExternalReference_（特别是_IfcClassificationReference_或_IfcLibraryReference_）与轮廓关联。

_IfcClassificationReference_用于引用轮廓规范（通用标准或制造商标准）。在这种情况下：

  • _IfcClassificationReference.ItemReference_包含规范中的正式轮廓名称。（另一方面，_IfcProfileDef.ProfileName_包含可显示名称，可能与正式名称不同。）
  • _IfcClassificationReference.Name_携带轮廓规范的简称。
  • 可选地，可以通过_IfcClassificationReference.ReferencedSource_进一步描述规范。

_IfcLibraryReference_用于引用包含轮廓定义的库。在这种情况下：

  • _IfcLibraryReference.ItemReference_包含库中轮廓的标识符，旨在机器可读（与应人类可读的_IfcProfileDef.ProfileName_相对）。
  • _IfcLibraryReference.Location_和._Name_或._ReferencedLibrary_进一步描述库。

如果提供了外部引用，发送系统应确保轮廓定义对象的形状与引用的分类或库中的定义一致。

_IfcProfileDef_的直接实例

通常，只应实例化_IfcProfileDef_的子类型。在某些特殊情况下，例如，如果轮廓对象用于几何模型以外的目的（如结构分析模型），可以直接实例化_IfcProfileDef_，并仅通过外部引用或轮廓属性进一步指定轮廓。后者通过逆属性_HasProperties_跟踪。

8.15.3.17.5 概念用法（Concept usage）

8.15.3.17.6 形式化表示（Formal representation） 

ENTITY IfcProfileDef
 SUPERTYPE OF (ONEOF
	(IfcArbitraryClosedProfileDef
	,IfcArbitraryOpenProfileDef
	,IfcCompositeProfileDef
	,IfcDerivedProfileDef
	,IfcOpenCrossProfileDef
	,IfcParameterizedProfileDef));
	ProfileType : IfcProfileTypeEnum;
	ProfileName : OPTIONAL IfcLabel;
 INVERSE
	HasExternalReference : SET [0:?] OF IfcExternalReferenceRelationship FOR RelatedResourceObjects;
	HasProperties : SET [0:?] OF IfcProfileProperties FOR ProfileDefinition;
END_ENTITY;
