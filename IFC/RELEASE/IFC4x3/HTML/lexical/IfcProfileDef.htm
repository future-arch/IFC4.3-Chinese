8.15.3.17 IfcProfileDef

RV ⓘ
AbV ⓘ

8.15.3.17.1 语义定义（Semantic definition）

IfcProfileDef 是 IFC 中所有标准和任意轮廓定义的超类型。它用于通过其参数或其显式曲线几何定义一组标准的常用截面轮廓。

  • 参数化轮廓是二维图元，在行业中用于通过其参数描述来描述截面。
  • 任意轮廓是截面，由作为有界曲线的外部边界定义，该曲线也可能包含由内部边界定义的孔。
  • 基于父轮廓变换的派生轮廓也是可用的轮廓定义的一部分。
  • 此外，还可以定义复合轮廓，其中包含两个或多个轮廓定义来定义最终轮廓。
沿革 New entity in IFC1.5, the capabilities have been extended in IFC2x. Profiles can now support swept surfaces and swept area solids with inner boundaries. It had been renamed from IfcAttDrivenProfileDef.
IFC4 变更 Changed from ABSTRACT to non-abstract for uses which do not require an explicitly defined geometry. Added inverse attributes HasProperties and HasExternalReference.

在材料关联中的使用

梁、柱以及形状相似的建筑构件及其类型对象可以通过 IfcRelAssociatesMaterial 与 IfcMaterialProfileSet 和 IfcMaterialProfileSetUsage 一起关联到截面轮廓定义，并结合材料定义。这样，具有相同截面和材料的建筑构件和构件类型可以共享一个通用的截面轮廓定义和关联。

材料关联中的轮廓定义必须与相应建筑构件的形状表示一致。

可以通过 IfcMaterialProfileSetUsage 中的基点参考来提供成员截面轮廓空间对齐的更高级别描述（例如居中、左下、几何重心等）。这可以与几何数据冗余使用，以传达设计意图。

在形状模型中的使用

轮廓定义用于几何和几何模型资源，以创建扫掠曲面、扫掠面积实体或分段脊。

轮廓定义在扫掠曲面或扫掠面积实体中的目的是定义一个被扫掠的均匀截面：

  • 沿直线（挤出）使用 IfcSurfaceOfLinearExtrusion 或 IfcExtrudedAreaSolid
  • 沿圆弧（旋转）使用 IfcSurfaceOfRevolution 或 IfcRevolvedAreaSolid
  • 沿位于参考曲面上的母线使用 IfcSurfaceCurveSweptAreaSolid

轮廓定义在分段脊中的目的是在脊曲线的多个位置定义变化的截面。子类型 IfcDerivedProfileDef 特别适合提供连续轮廓，这些轮廓基于起始轮廓的变换，从而保持顶点和边的同一性。

注释 Subtypes of the IfcProfileDef contain parameterized profiles (as subtypes of IfcParameterizedProfileDef) which establish their own 2D position coordinate system, profiles given by explicit curve geometry (either open or closed profiles) and two special types for composite profiles and derived profiles, based on a 2D Cartesian transformation.

当 IfcProfileDef 用于扫掠面积实体时，它被视为有界区域。在这种情况下，轮廓的内部是轮廓的一部分。ProfileType 属性设置为 AREA。当 IfcProfileDef 用于扫掠曲面时，它被视为曲线。在这种情况下，轮廓的内部（如果曲线是封闭的）不是轮廓的一部分。ProfileType 属性设置为 CURVE。

图1和图2说明了在扫掠面积实体中使用参数化轮廓。

IfcProfileDef 在底层坐标系中定义，该坐标系由使用轮廓定义的扫掠曲面或扫掠面积实体定义。它是：

  • IfcSweptSurface.Position 或
  • IfcSweptAreaSolid.Position 或
  • IfcSectionedSpine.CrossSectionPositions 的每个列表成员的 xy 平面。

在图1中，左侧的位置坐标系的 z 轴指向绘图平面之外。

注释 The subtype IfcParameterizedProfileDef optionally provides an additional 2D position coordinate system relative to the underlying coordinate system of the IfcProfileDef.

在 IfcProfileDef 在扫掠曲面或扫掠面积实体中的后续使用中，例如 IfcExtrudedAreaSolid（此处用作示例），轮廓边界（此处基于 IfcParameterizedProfileDef 的二维位置坐标系）放置在扫掠曲面或扫掠面积实体的三维位置坐标系的 xy 平面内。

轮廓通过以下方式插入到底层坐标系中：

  • 在使用 IfcArbitraryClosedProfileDef 和 IfcArbitraryOpenProfileDef 时直接插入，
  • 在使用 IfcParameterizedProfileDef 时通过中间位置坐标系插入。
  • 在使用 IfcDerivedProfileDef 时通过二维笛卡尔变换算子插入（直接应用于使用任意轮廓定义时的曲线位置，或应用于使用参数化轮廓定义时的位置坐标系）。
  • 使用 IfcCompositeProfileDef 时，插入取决于包含的子轮廓的子类型。

轮廓类型

此处演示了 ProfileType 属性不同用法的效果。ProfileType 定义了轮廓定义是解析为曲线还是曲面区域。在 AREA 的情况下，轮廓的内部（有界区域）是轮廓定义的一部分。在 CURVE 的情况下，轮廓的内部（如果曲线是封闭的）不是轮廓的一部分。图3和图4说明了根据 ProfileType 产生的区域或曲线。

通过外部引用指定轮廓

如果轮廓由某个规范或目录标准化，则应通过 HasExternalReference 提供指向该规范或目录的引用。此逆向关系用于将 IfcExternalReference（特别是 IfcClassificationReference 或 IfcLibraryReference）与轮廓关联。

IfcClassificationReference 用于引用轮廓规范（通用标准或制造商标准）。在这种情况下，

  • IfcClassificationReference.ItemReference 包含来自规范的正式轮廓标识。 （另一方面，IfcProfileDef.ProfileName 包含一个可显示的名称，该名称不一定与正式标识相同。）
  • IfcClassificationReference.Name 包含轮廓规范的简称。
  • 可选地，可以通过 IfcClassificationReference.ReferencedSource 进一步描述规范。

IfcLibraryReference 用于引用包含轮廓定义的库。在这种情况下，

  • IfcLibraryReference.ItemReference 包含库中轮廓的标识符，并且是机器可读的（与 IfcProfileDef.ProfileName 不同，后者应为人类可读）。
  • IfcLibraryReference.Location 和 .Name 或 .ReferencedLibrary 进一步描述库。

如果提供了外部引用，发送系统应确保轮廓定义对象的形状与引用的分类或库中的定义一致。

IfcProfileDef 的直接实例

通常，应仅实例化 IfcProfileDef 的子类型。在某些特殊情况下，例如，如果轮廓对象用于几何模型以外的目的（例如，用于结构分析模型），则可能可以直接实例化 IfcProfileDef，并且仅通过外部引用或轮廓属性来进一步指定轮廓。后者由逆向属性 HasProperties 进行跟踪。

8.15.3.17.5 概念用法（Concept usage）

8.15.3.17.6 形式化表示（Formal representation） 

ENTITY IfcProfileDef
 SUPERTYPE OF (ONEOF
	(IfcArbitraryClosedProfileDef
	,IfcArbitraryOpenProfileDef
	,IfcCompositeProfileDef
	,IfcDerivedProfileDef
	,IfcOpenCrossProfileDef
	,IfcParameterizedProfileDef));
	ProfileType : IfcProfileTypeEnum;
	ProfileName : OPTIONAL IfcLabel;
 INVERSE
	HasExternalReference : SET [0:?] OF IfcExternalReferenceRelationship FOR RelatedResourceObjects;
	HasProperties : SET [0:?] OF IfcProfileProperties FOR ProfileDefinition;
END_ENTITY;
