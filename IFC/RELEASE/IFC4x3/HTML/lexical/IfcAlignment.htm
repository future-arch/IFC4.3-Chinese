5.4.3.1 IfcAlignment（公路路线）

注释 该实体是 IfcProduct 或 IfcTypeProduct 的子类型，因此是每个标准化模式子集和实现级别的一部分。

5.4.3.1.1 语义定义（Semantic definition）

在IFC标准中，英文术语"alignment"定义了三个独立但紧密关联的概念。

  1. 线性定位参考系统的定义
  2. 车辆移动的安全保障与优化 - 运动学视角
  3. 道路、铁路轨道或其他线性基础设施的几何构造

线性定位参考系统

公路路线用于定义参考系统，主要用于线性工程（如道路、轨道、桥梁等）中元素的定位。沿公路路线的相对定位通过线性参考方法定义。

注释 关于线性参考的一般定义，请参见ISO 19148地理信息-线性参考。

运动学视角

在运动学视角中，重点是在水平和垂直布局方向变化引起的约束条件下，实现车辆的安全和优化移动。

几何视角

在几何视角中，重点是正确布置水平和垂直段以连接拟建路径上的特定点。这方面的知识体系经过长期发展，在许多方面早于现代计算机及其软件的出现。

IfcAlignment 支持的形状表示包括：

  • IfcCompositeCurve 作为2D水平路线（由其水平路线段表示），无垂直布局。
  • IfcGradientCurve 作为3D水平和垂直路线（由其路线段表示），无超高布局。
  • IfcSegmentedReferenceCurve 作为相对于 IfcGradientCurve 定义的3D曲线，用于纳入超高应用。
  • IfcOffsetCurveByDistances 作为相对于 IfcGradientCurve 或其他 IfcOffsetCurveByDistances 定义的2D或3D曲线。
  • IfcPolyline 或 IfcIndexedPolyCurve 通过3D多段线表示作为3D路线（如来自测量数据）。
  • IfcPolyline 或 IfcIndexedPolyCurve 通过2D多段线表示作为2D水平路线（如在非常早期的规划阶段或作为地图表示）。

当代工程的技术现状 通常，当代工程在定义公路路线时遵循以下步骤：

  1. 首先，在适当投影平面中建立水平布局。

  2. 第二步，添加垂直布局（即具有恒定坡度的段序列，以及显示坡度变化的平滑段）。

  3. 在铁路领域，大多数情况下会在水平布局上添加超高布局，以补偿部分不良横向加速度。

  4. 最后一步，根据定义的规则、公式和阈值检查拟议布局，以确保符合规范要求。

这些步骤的顺序可能因具体情况而异，并且可能重复一次或多次以实现经济目标并满足监管安全要求。

当代公路路线设计几乎总是采用2.5维方法。最终记录在案的几何可能非常精确，或者仅满足安全阈值要求。这取决于管理优先级、设计日期（现有路线可能是在50多年前设计的）或使用的软件工具等因素。在高精度BIM模型中处理遗留数据需要充分理解这些因素。

业务逻辑与几何定义的区别

公路路线概念分为两个部分。这两个部分协同工作，但也可以独立使用和交换。

  1. 公路路线的业务逻辑
  2. 公路路线的几何定义

业务逻辑： IFC模式允许使用尽可能接近业务术语和概念的方式来描述公路路线。它允许描述构成公路路线的布局（即水平、垂直、超高）、它们的段结构和属性。此外，业务逻辑部分为领域特定属性（如设计速度或超高不足）提供了锚定点。

几何定义： IFC模式提供了完善的IFC几何实体来表示业务概念。

业务逻辑与其在IFC中的几何定义之间的映射通过公路路线几何相关的概念模板进行描述。
