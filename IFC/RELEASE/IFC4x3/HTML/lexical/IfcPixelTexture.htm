8.12.3.22 IfcPixelTexture

注释 该实体不是标准化模式子集或实现级别的一部分。

8.12.3.22.1 语义定义（Semantic definition）

IfcPixelTexture 提供了一个基于图像的二维纹理图，作为像素值（Pixel 二进制属性列表）的显式数组。与 IfcImageTexture 不同，IfcPixelTexture 直接保存像素颜色（和不透明度）的二维列表，而不是引用 URL。

以下来自 ISO/IEC 19775-1 X3D 体系结构和基本组件（X3D 规范）的定义适用：

  • PixelTexture 节点定义了一个基于图像的二维纹理图，作为像素值（图像字段）的显式数组，以及控制纹理在几何体上平铺重复的参数。
  • 纹理图在二维坐标系 (s, t) 中定义，该坐标系在两个方向上范围从 0.0 到 1.0。像素图像的底边对应于纹理图的 S 轴，像素图像的左边对应于纹理图的 T 轴。像素图像的左下像素对应于 s=0.0, t=0.0，图像的右上像素对应于 s = 1.0, t = 1.0。
  • Image 字段指定单个未压缩的二维像素图像。图像字段包含三个整数，分别代表图像的宽度、高度和组件数，后跟宽度×高度的十六进制值，表示图像中的像素。像素值限制为 256 级强度（即十六进制 0x00-0xFF）。
  • 单组件图像指定一个字节的十六进制值，表示图像的强度。例如，0xFF 是十六进制中的全强度（十进制中的 255），0x00 是无强度（十进制中的 0）。
  • 双组件图像指定第一个字节（高位）的强度和第二个字节（低位）的 Alpha 不透明度。
  • 三组件图像中的像素指定第一个字节（高位）的红色分量，然后是绿色和蓝色分量（例如，0xFF0000 是红色，0x00FF00 是绿色，0x0000FF 是蓝色）。
  • 四组件图像在红/绿/蓝之后指定 Alpha 不透明度字节（例如，0x0000FF80 是半透明蓝色）。值为 00 表示完全透明，FF 表示完全不透明，80 表示半透明。
  • 请注意，如果 Alpha 和透明度都从 0.0 到 1.0 变化，则 Alpha = (1.0 - 透明度)。

8.12.3.22.5 形式化表示（Formal representation） 

ENTITY IfcPixelTexture
 SUBTYPE OF (IfcSurfaceTexture);
	Width : IfcInteger;
	Height : IfcInteger;
	ColourComponents : IfcInteger;
	Pixel : LIST [1:?] OF IfcBinary;
 WHERE
	MinPixelInS : Width >= 1;
	MinPixelInT : Height >= 1;
	NumberOfColours : {1 <= ColourComponents <= 4};
	PixelAsByteAndSameLength : SIZEOF(QUERY(temp<* Pixel |
 (BLENGTH(temp) MOD 8 = 0) AND
 (BLENGTH(temp) = BLENGTH(Pixel[1]))
)) = SIZEOF(Pixel);
	SizeOfPixelList : SIZEOF(Pixel) = (Width * Height);
END_ENTITY;
