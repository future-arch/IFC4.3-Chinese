5.4.3.54 IfcRelInterferesElements（关系干涉构件）

AbV ⓘ

5.4.3.54.1 语义定义（Semantic definition）

IfcRelInterferesElements 对象化关系表示两个构件之间存在干涉。

这是一种一对一关系，两个构件在物理或逻辑上相互干涉的概念独立于构件本身进行描述。

可通过提供干涉几何体或区域的方式，将干涉与实体的形状表示相关联：

  • 当干涉发生在两个物理产品之间时，使用 InterferenceGeometry 属性定义物理干涉形状，它可以是关系中使用的某个构件形状的一部分，也可以使用 IfcConnectionGeometry 定义两个形状之间的接口。
  • 当干涉发生在两个空间对象之间时，使用 InterferenceSpace 属性定义与空间对象相关联的两个覆盖区之间的干涉空间，通过预定义类型为 INTERFERENCE 的特定 IfcSpatialZoneTypeEnum 的 IfcSpatialZone 来表达。

如果省略干涉几何体和区域，则干涉以逻辑关系的形式提供。在这种情况下，连接点、曲线、曲面或实体必须由接收应用程序重新计算。

RelatingElement 和 RelatedElement 定义了关系中的两个构件，它们可能具有不同的角色。 某些 InterferenceType 值或 InterferenceGeometry 计算可能需要关系方向，这通过相应设置 ImpliedOrder 属性来实现：

  • ImpliedOrder=TRUE RelatingElement 构成干涉关系的主要构件，方向从 RelatingElement（源）指向 RelatedElement（目标）。如果通过减去重叠部分来解决干涉，则应从 RelatingElement 中减去。最终结果将是 RelatingElement 减去 InterferenceGeometry。这适用于 RelatedElement 在 RelatingElement 中动态创建空腔的干涉关系。
  • ImpliedOrder=FALSE RelatingElement 和 RelatedElement 彼此之间没有优先级。如果要解决干涉，则无法追溯 InterferenceGeometry 应从 RelatingElement 还是 RelatedElement 中减去的信息。这适用于碰撞检测结果。
  • ImpliedOrder=UNKNOWN 未提供有关优先级的信息。

InterferenceType 属性可选地指定两个构件之间的干涉类型，提供了两组默认类型：

  • 定向干涉类型 暗示使用设置为 TRUE 的 ImpliedOrder，并且需要特定选择 RelatingElement 和 RelatedElement 以使其有意义：
  • Crosses：RelatingElement 穿过 RelatedElement（例如，铁路穿过道路）
  • PassesThrough：RelatingElement 穿过 RelatedElement（例如，道路穿过隧道内部）
  • PassesOver：RelatingElement 从 RelatedElement 上方通过（例如，桥梁从水渠上方通过）
  • PassesUnder：RelatingElement 从 RelatedElement 下方通过（例如，隧道从道路下方通过）
  • 非定向干涉类型 不暗示特定的 ImpliedOrder 值（但仍可设置以详细说明形状干涉计算）
  • Clash：RelatingElement 和 RelatedElement 存在基于空间或形状的碰撞
  • Along：RelatingElement 和 RelatedElement 具有共同的边界/曲面

5.4.3.54.6 形式化表示（Formal representation） 

ENTITY IfcRelInterferesElements
 SUBTYPE OF (IfcRelConnects);
	RelatingElement : IfcInterferenceSelect;
	RelatedElement : IfcInterferenceSelect;
	InterferenceGeometry : OPTIONAL IfcConnectionGeometry;
	InterferenceType : OPTIONAL IfcIdentifier;
	ImpliedOrder : IfcLogical;
	InterferenceSpace : OPTIONAL IfcSpatialZone;
 WHERE
	NoSelfReference : RelatingElement :<>: RelatedElement;
END_ENTITY;
