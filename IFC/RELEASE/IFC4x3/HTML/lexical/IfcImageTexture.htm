8.12.3.17 IfcImageTexture

RV ⓘ
AbV ⓘ

8.12.3.17.1 语义定义（Semantic definition）

IfcImageTexture 提供了一个二维纹理，可以应用于几何项的表面，并提供其映射到的表面的照明参数。纹理作为图像文件位于外部位置，并提供其 URL。

ISO/IEC 19775-1 X3D 体系结构和基本组件（X3D 规范）中的以下定义适用：

  • 具有透明度或简单透明度的灰度像素被视为强度纹理。
  • 具有透明度或简单透明度的灰度像素被视为强度加透明度纹理。
  • 没有透明度通道或简单透明度的 RGB 像素被视为完整 RGB 纹理。
  • 具有透明度通道或简单透明度的 RGB 像素被视为完整 RGB 加透明度纹理。
  • 如果图像指定颜色为索引颜色（即调色板或颜色图），则应使用以下语义（请注意，`greyscale' 指的是具有相等红、绿、蓝值的调色板条目）：
  • 如果调色板中的所有颜色都是灰度色且没有透明度块，则将其视为强度纹理。
  • 如果调色板中的所有颜色都是灰度色且存在透明度块，则将其视为强度加不透明度纹理。
  • 如果调色板中的任何颜色不是灰色且没有透明度块，则将其视为完整 RGB 纹理。
  • 如果调色板中的任何颜色不是灰色且存在透明度块，则将其视为完整 RGB 加透明度纹理。
  • 需要支持 JPEG 文件的纹理节点应按以下方式解释 JPEG 文件：
  • 灰度文件（组件数为 1）被视为强度纹理。
  • YCbCr 文件被视为完整 RGB 纹理。
  • 不需要其他 JPEG 文件类型。建议将其他 JPEG 文件视为完整 RGB 纹理。
  • 建议支持 GIF 文件的纹理节点应遵循上述适用于 PNG 格式的语义。

统一资源定位符 (URL) 是 URI 的一种形式，由 IETF 在 RFC1738 中定义。它支持通过特定协议（通常是 http）访问特定服务器上的资源，以及访问本地机器上的资源。

注释 遵循 ifcZIP 约定的交换文件可能包含图像资源的子目录结构，以便与产品数据集一起存储。
注释 此标准中的纹理定义是根据 X3D 的纹理组件开发的。请参阅 ISO/IEC 19775-1.2:2008 X3D 体系结构和基本组件 第二版，第 1 部分，18 纹理组件，了解国际标准中的定义。
沿革 IFC2x2 新增实体。
