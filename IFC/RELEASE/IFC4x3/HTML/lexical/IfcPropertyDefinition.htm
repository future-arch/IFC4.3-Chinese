5.1.3.13 IfcPropertyDefinition（属性定义）

RV ⓘ
AbV ⓘ

抽象 该定义可能不会被直接实例化。

5.1.3.13.1 语义定义（Semantic definition）

IfcPropertyDefinition 定义了可分配给对象的所有特征（即单个属性的分组）的泛化。目前，IfcPropertyDefinition 的子类型包括属性集实例、属性集模板和属性模板。

  • 属性集模板 - IfcPropertySetTemplate，一组属性模板的集合，用于确定项目上下文中使用的属性定义。
  • 属性模板 - IfcPropertyTemplate，一个单独的模板，用于确定同一项目上下文中使用的特定属性的定义。该模板可以确定基于该模板的每个属性的名称、描述、数据类型、单位或标准表达式。
  • 属性集实例 - IfcPropertySet，一组单个属性（可能由属性模板确定，也可能不由属性模板确定）的集合，包含单个值、度量类型和单位，并与对象实例或对象类型相关联。
注释 IfcPropertyDefinition 的子类型层次结构还包括静态定义的属性集，如 IfcPreDefinedPropertySet。这些是在实体定义中组合的固定属性的很少使用的集合。IfcPreDefinedPropertySet 不能由 IfcPropertySetTemplate 确定。
注释 单个属性（IfcProperty 的子类型）目前未包含在 IfcPropertyDefinition 的子类型层次结构中。这种异常是由于与本标准早期版本的向上兼容性原因造成的。

属性定义定义了在对象的多个实例（无论是对象实例还是对象类型）之间共享的信息。IfcPropertyDefinition（通过其可实例化的子类型）可以参与以下关系：

  • 分配给项目上下文 - 与 IfcRelDeclares 的 HasContext 关系，用于建立声明此属性定义的项目上下文。此关系主要适用于 IfcPropertyTemplateDefinition 的子类型。
  • 与外部资源的关联 - 与 IfcRelAssociates 的 HasAssociations 关系，该关系引用外部信息源（最显著的是分类或文档）并创建单向关联。关联不隐含任何依赖关系。

子类型包含在更具体的关系中，详见 IfcPropertySetDefinition 和 IfcPropertyTemplateDefinition。

5.1.3.13.5 形式化表示（Formal representation） 

ENTITY IfcPropertyDefinition
 ABSTRACT SUPERTYPE OF (ONEOF
	(IfcPropertySetDefinition
	,IfcPropertyTemplateDefinition))
 SUBTYPE OF (IfcRoot);
 INVERSE
	HasContext : SET [0:1] OF IfcRelDeclares FOR RelatedDefinitions;
	HasAssociations : SET [0:?] OF IfcRelAssociates FOR RelatedObjects;
END_ENTITY;
