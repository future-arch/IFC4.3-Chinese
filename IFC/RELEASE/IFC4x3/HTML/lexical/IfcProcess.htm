5.1.3.9 IfcProcess

RV ⓘ
AbV ⓘ

抽象 该定义可能不会被直接实例化。

5.1.3.9.1 语义定义（Semantic definition）

IfcProcess 定义为在时间上排序的单个活动或事件，与其他流程具有顺序关系，将输入转换为输出，并可能通过输入输出关系连接到其他流程。_IfcProcess_可以是活动（或任务）或事件。它通常发生在建筑施工中，目的是设计、成本计算、采购、建造或维护产品或其他类似任务或程序。图1说明了流程关系。

注意 根据ISO9000的定义： 流程是一组相互关联或相互作用的活动的集合。流程使用资源将输入转化为输出。流程是相互连接的，因为一个流程的输出成为另一个流程的输入。实际上，流程通过这种输入输出关系"粘合"在一起。

流程信息通过建立以下关系与其他对象相关：

  • 流程嵌套：IfcRelNests - 一个流程可以包含子流程，从而被嵌套。
  • 流程排序：IfcRelSequence - 流程可以按顺序放置（包括并行任务的重叠），并具有前驱和后继。
  • 将流程分配给计划：IfcRelAssignsToControl - 活动（如任务，主要是摘要任务）被分配给工作计划。
  • 将产品作为输入分配给流程：IfcRelAssignsToProcess - 产品可以作为输入分配给流程，例如用于施工流程规划。
  • 将产品作为输出分配给流程：IfcRelAssignsToProduct - 产品可以作为输出分配给流程，例如用于施工流程规划。
  • 将控制作为流程控制分配给流程：IfcRelAssignsToProcess - 作为流程控制的项目可以分配给流程，例如用于成本管理（将成本项分配给工作任务）。
  • 将资源作为流程消耗分配给流程：IfcRelAssignsToProcess - 作为流程机制的项目，如成本计算中的劳动力、材料和设备。
沿革 IFC1.0中的新实体。

IFC2x变更 已移除_Productivity_属性。

IFC4变更 _Identification_属性已从子类型_IfcTask_和其他类型提升。

5.1.3.9.5 概念用法（Concept usage）

5.1.3.9.6 形式化表示（Formal representation） 

ENTITY IfcProcess
 ABSTRACT SUPERTYPE OF (ONEOF
	(IfcEvent
	,IfcProcedure
	,IfcTask))
 SUBTYPE OF (IfcObject);
	Identification : OPTIONAL IfcIdentifier;
	LongDescription : OPTIONAL IfcText;
 INVERSE
	IsPredecessorTo : SET [0:?] OF IfcRelSequence FOR RelatingProcess;
	IsSuccessorFrom : SET [0:?] OF IfcRelSequence FOR RelatedProcess;
	OperatesOn : SET [0:?] OF IfcRelAssignsToProcess FOR RelatingProcess;
END_ENTITY;
